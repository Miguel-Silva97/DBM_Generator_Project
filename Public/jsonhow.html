Informação Necessária Para Editar JSON Schemas Na Plataforma

Projeto desenvolvido no âmbito da UC de Desenvolvimento Baseado a Modelos, com o objetivo que gerar um sistema e o seu backoffice/base de dados associada através de um ficheiro JSON com um JSON Schema, começe por gerar o sistema ou editar os ficheiros na barra de navegação escolha ficheiros JSON

Title, Type e Description

Title - Titulo da classe que dará nome ao JSON Schema, só aceita e letras e começar por letra Maiuscula

Type - O tipo deverá ser sempre "object"

Description - Uma breve descrição do que representa a classe

Properties

Onde se define cada variavel sendo que o nome deve ser só letras em minusculas

Description- Breve descrição do que representa o atributo

PrettyName- Nome da variável mais apresentável para ser apresentado nas views Exemplo: releaseDate > Data de Lançamento

Type - Os tipos de variável aceite pelo sistema são "string","date","integer" e "number"

Restrições:

Unico: Se pretender uma variavel de valor unico definir "unique" = true, é falso por omissao

Maximo ou Minimo(number,integer): Sempre valores acima de 0 e inteiros, se pretender maximo "maximum" = 2, se pretender minimo "minimum" = 1

Tamanho Maximo ou Minimo(string): Sempre valores acima de 0 e inteiros, se pretender maximo "maxLength" = 254, se pretender minimo "minLength" = 20

Required

Neste array de parametros, deve colocar as variáveis que criou para a classe que pretende que sejam obrigatórias

References

Se pretender adicionar uma referencia ou varias de uma tabela adicionar o parametro references neste formato:

"references": [ { "model": "Artist", "relation": "1-M", "label": "name" } ]

Model - Classe que está a referenciar

Relation - Tipo de relação entre as classes 1-M ,M-M ou 1-1

Label - Variável da classe que está a ser referenciada que quer utilizar para demonstrar a relação
